# AD 密码管理系统 - 后台启动使用说明

## 📋 概述

系统提供两种启动方式:
- **前台启动**: 适合开发调试，可以看到实时日志
- **后台启动**: 适合生产环境，服务在后台运行

## 🚀 启动脚本说明

### 1. 前台启动脚本 (开发调试用)

**文件名**: `start_windows.bat`

**特点**:
- ✅ 显示实时日志
- ✅ 可以看到详细的运行信息
- ❌ 不能关闭命令行窗口
- ❌ 适合临时测试

**使用场景**:
- 开发环境调试
- 查看系统运行日志
- 临时启动测试

### 2. 后台启动脚本 (生产环境用) ⭐

**文件名**: `start_windows_background.bat`

**特点**:
- ✅ 服务在后台运行
- ✅ 可以关闭命令行窗口
- ✅ 适合长期运行
- 📝 日志保存到文件

**使用场景**:
- 生产环境部署
- 需要长期运行
- 不需要实时查看日志

## 🛠️ 辅助工具

### 停止服务脚本

**文件名**: `stop_service.bat`

**功能**: 停止后台运行的 Flask 服务

**使用方法**:
```bash
# 双击运行或在命令行执行
stop_service.bat
```

### 状态检查脚本

**文件名**: `check_status.bat`

**功能**:
- ✅ 检查服务是否运行
- ✅ 检查端口 5000 占用情况
- ✅ 查看最近日志

**使用方法**:
```bash
# 双击运行或在命令行执行
check_status.bat
```

## 📖 使用流程

### 方式一：后台启动 (推荐)

```bash
# 1. 启动服务
start_windows_background.bat

# 2. 系统会提示是否打开浏览器
# 输入 Y 立即访问系统

# 3. 服务将一直在后台运行
# 可以关闭启动窗口

# 4. 需要停止时
stop_service.bat
```

### 方式二：前台启动

```bash
# 1. 启动服务
start_windows.bat

# 2. 保持命令行窗口打开
# 可以看到实时日志

# 3. 需要停止时
# 按 Ctrl+C 停止服务
```

## 🔍 常见问题

### Q1: 如何查看服务是否在运行？

**A**: 运行 `check_status.bat` 查看服务状态

### Q2: 后台启动后在哪里看日志？

**A**: 日志文件位置：`logs\app.log`

可以使用以下命令查看:
```bash
# 查看最近 10 条日志
powershell -Command "Get-Content logs\app.log -Tail 10"

# 实时查看日志
powershell -Command "Get-Content logs\app.log -Wait -Tail 5"
```

### Q3: 如何彻底停止服务？

**A**: 
1. 运行 `stop_service.bat`
2. 如果失败，打开任务管理器，结束 `python.exe` 进程

### Q4: 开机自动启动怎么办？

**A**: 有三种方法:

**方法 1**: 使用任务计划程序
1. 打开"任务计划程序"
2. 创建基本任务
3. 触发器选择"计算机启动时"
4. 操作选择"启动程序"
5. 程序选择 `start_windows_background.bat`

**方法 2**: 添加到启动文件夹
1. 创建快捷方式到 `start_windows_background.bat`
2. 将快捷方式复制到启动文件夹:
   `shell:startup`

**方法 3**: 使用第三方工具
- 使用 AutoStart Manager 等工具管理开机启动

### Q5: 端口 5000 被占用怎么办？

**A**:
1. 运行 `check_status.bat` 查看占用端口的进程
2. 停止占用端口的进程
3. 或者修改 `backend\.env` 文件中的端口配置

## 📝 日志管理

### 日志位置
- 主日志：`logs\app.log`
- 后端日志：`backend\logs\app.log`

### 查看日志
```bash
# 查看最近 20 条
powershell -Command "Get-Content logs\app.log -Tail 20"

# 实时查看
powershell -Command "Get-Content logs\app.log -Wait"

# 搜索特定内容
powershell -Command "Select-String -Path logs\app.log -Pattern 'ERROR'"
```

### 清理日志
```bash
# 清空日志文件
powershell -Command "Clear-Content logs\app.log"
```

## 🎯 最佳实践

1. **开发环境**: 使用 `start_windows.bat` 前台启动，方便调试
2. **生产环境**: 使用 `start_windows_background.bat` 后台启动
3. **定期检查**: 使用 `check_status.bat` 检查服务状态
4. **日志管理**: 定期清理或归档日志文件
5. **停止服务**: 优先使用 `stop_service.bat`，避免强制结束进程

## 📞 技术支持

如遇到问题，请检查:
1. Python 环境是否正确安装
2. Node.js 环境是否正确安装
3. 端口 5000 是否被占用
4. 日志文件中的错误信息
