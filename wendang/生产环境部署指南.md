## 🗄️ 数据库初始化

### PostgreSQL

```sql
-- 创建数据库
CREATE DATABASE ad_password_db;

-- 创建用户
CREATE USER aduser WITH PASSWORD 'YourStrongP@ssw0rd';

-- 授权
GRANT ALL PRIVILEGES ON DATABASE ad_password_db TO aduser;
```

### MySQL

```sql
-- 创建数据库
CREATE DATABASE ad_password_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'aduser'@'localhost' IDENTIFIED BY 'YourStrongP@ssw0rd';

-- 授权
GRANT ALL PRIVILEGES ON ad_password_db.* TO 'aduser'@'localhost';
FLUSH PRIVILEGES;
```

### SQL Server

```sql
-- 创建数据库
CREATE DATABASE ad_password_db;

-- 创建登录
CREATE LOGIN aduser WITH PASSWORD = 'YourStrongP@ssw0rd';

-- 创建用户
USE ad_password_db;
CREATE USER aduser FOR LOGIN aduser;

-- 授权
ALTER ROLE db_owner ADD MEMBER aduser;
```

## 🚀 启动服务

### 开发环境（不推荐生产使用）

```bash
cd backend
.venv\Scripts\activate  # Windows
# source .venv/bin/activate  # Linux

python app.py
```

### 生产环境 - Windows (使用 Waitress)

```bash
cd backend
.venv\Scripts\activate

waitress-serve --host=0.0.0.0 --port=5000 app:app
```

### 生产环境 - Linux (使用 Gunicorn)

```bash
cd backend
source .venv/bin/activate

gunicorn -w 4 -b 0.0.0.0:5000 app:app
```

### 使用 systemd 管理（Linux）

创建服务文件：

```ini
# /etc/systemd/system/ad-password.service
[Unit]
Description=AD Password Management System
After=network.target postgresql.service

[Service]
Type=notify
User=www-data
Group=www-data
WorkingDirectory=/path/to/ad2/backend
Environment="PATH=/path/to/ad2/backend/.venv/bin"
ExecStart=/path/to/ad2/backend/.venv/bin/gunicorn -w 4 -b 0.0.0.0:5000 app:app
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start ad-password
sudo systemctl enable ad-password
sudo systemctl status ad-password
```

## 🌐 Nginx 反向代理

### 1. 安装 Nginx

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install nginx
```

**CentOS/RHEL:**
```bash
sudo yum install epel-release
sudo yum install nginx
```

### 2. 配置 Nginx

```nginx
# /etc/nginx/sites-available/ad-password
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    
    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;
    
    # SSL 证书配置
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    # 安全头
    add_header Strict-Transport-Security "max-age=31540000; includeSubDomains" always;
    add_header X-Content-Type-Options nosniff always;
    add_header X-Frame-Options SAMEORIGIN always;
    add_header X-XSS-Protection "1; mode=block" always;
    
    # 前端静态文件
    location / {
        root /path/to/ad2/frontend/dist;
        try_files $uri $uri/ /index.html;
        
        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }
    
    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    # 日志
    access_log /var/log/nginx/ad-password-access.log;
    error_log /var/log/nginx/ad-password-error.log;
}
```

### 3. 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/ad-password /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```
